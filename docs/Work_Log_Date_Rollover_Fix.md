# 작업일보 날짜 드래그 오류 보완 리포트

## 문제 요약
`@압출일보 2호기(251223).xlsm`에서 2025-12-18 구간(행 2489~2505)에  
`주 -> 야 -> 주 -> 야` 패턴이 반복됩니다.  
이는 **야간 교대 이후 다음날 주간이 12/19로 넘어가야 하는데, 날짜를 12/18로 드래그 복사**한 오류입니다.  
결과적으로 시간 순서와 집계가 뒤섞입니다.

## 목표
- 12/18의 패턴은 **주 -> 야**로 끝나야 함
- 다음 주간부터는 **12/19로 날짜가 넘어가도록 자동 보정**

## 보완 로직(권장)
### 1) 기본 날짜 계산
`source_date`(엑셀 날짜)와 `start_time`(시각)을 사용하여 기본 날짜를 설정합니다.

규칙
- `start_time < 08:00`이면 **기본 날짜를 +1일**로 설정  
  (예: 12/17 00:20 → 12/18)
- 그 외는 `source_date` 그대로 사용

### 2) 날짜 롤오버 감지(드래그 오류 대응)
같은 `source_date`에서 **주 -> 야 -> 주** 패턴이 나오면,  
**두 번째 주간부터 날짜를 +1일**로 보정합니다.

보조 조건(시간 역전)
- `현재 start_time < 이전 start_time` 또는  
  `현재 start_time < 이전 end_time`인 경우 **날짜 +1일**

### 3) 종료 시간 보정
- 날짜 보정 후에도 `end_time < start_time`이면  
  **end_time에 +1일** 적용

## 구현 제안(업로더 로직)
### A안: 자동 보정(권장)
1) 엑셀 행 순서를 `source_row`로 저장  
2) `source_date + start/end 시각`으로 임시 시간 생성  
3) 행 순서 기반으로 롤오버 감지  
4) 보정된 날짜로 `start_time/end_time` 재계산  
5) 변경 행은 `data_quality_flags`에 기록

예시 플래그
```
{
  "date_corrected": true,
  "reason": "shift_rollover",
  "source_date": "2025-12-18"
}
```

### B안: 업로드 전 사용자 확인
업로드 UI에서 날짜 보정 감지 시  
자동 보정 여부를 사용자에게 선택하게 함

## 검증 지표
- 보정된 행 수 / 전체 행 수
- 보정 전후 `start_time` 단조 증가율
- `주 -> 야` 패턴 정상화 여부

## 결론
이번 문제는 **날짜 드래그 복사 오류**입니다.  
가장 안전한 해결책은 **기본 날짜 보정 + 주/야 패턴 롤오버 감지**를 업로더에 반영하는 것입니다.
